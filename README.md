# Beevolution

**Beevolution** — это интерактивный симулятор эволюции пчелиного роя на базе генетического алгоритма (ГА), разработанный на Python с использованием библиотек Pygame и Numba. Проект создан в образовательных целях и наглядно демонстрирует принципы естественного отбора, наследования и мутаций.

Главная цель роя — найти путь к мишени вверху экрана. Игрок может мешать пчелам, рисуя защитные стены мышкой прямо во время симуляции, или выбирая готовые преграды-лабиринты. Пчелы при этом адаптируются и эволюционируют, обучаясь облетать новые препятствия.

## Ключевые особенности

* **Интерактивное обучение:** Встроенный пошаговый туториал объясняет ключевые понятия генетики (ДНК, хромосомы, естественный отбор, мутации) понятным языком.
* **Физика на базе JIT-компиляции:** Расчет движения, столкновений со стенами и физики сдвига объектов оптимизирован с помощью библиотеки **Numba**, что обеспечивает стабильную производительность при размере популяции в 200 особей.
* **Умная функция приспособленности (Fitness Function):** Вместо простого евклидова расстояния по прямой, в алгоритме используется волновой алгоритм (BFS) для расчета реальной длины пути в обход стен. Это позволяет популяции успешно решать сложные пространственные задачи и проходить лабиринты.
* **Интерактивность:** Возможность рисовать стены мышкой на лету, регулировать силу мутаций кнопками, а также перематывать поколения на высокой скорости.

## Как это работает с точки зрения биологии

1. **Генотип (ДНК):** Каждая пчела рождается со своей хромосомой, состоящей из 500 векторов силы (импульсов движения), которые она последовательно применяет на каждом шаге жизни. В первом поколении ДНК заполняется случайно.
2. **Естественный отбор (Селекция):** По окончании жизненного цикла (500 шагов) каждая особь оценивается функцией приспособленности. Пчелы, которые подобрались ближе к цели по свободному пути, получают больше очков.
3. **Кроссинговер (Скрещивание):** 20% наиболее успешных особей («элита») становятся родителями для следующего поколения. Их ДНК смешивается, передавая потомкам удачные маршруты.
4. **Мутация:** При копировании ДНК у потомков с определенной вероятностью (регулируется игроком) происходят случайные изменения векторов. Это позволяет открывать принципиально новые траектории движения в обход свежепостроенных стен.

## Управление

### Мышь
* **Левая кнопка мыши (ЛКМ):** Зажмите и проведите линию на игровом поле, чтобы построить стену.

### Клавиатура
* `Стрелка ВВЕРХ` / `Стрелка ВНИЗ` — увеличить / уменьшить вероятность мутаций в популяции.
* `Клавиши [1] - [5]` — загрузка предустановленных испытаний и лабиринтов разной сложности.
* `Клавиша [C]` — очистить игровое поле от всех стен.
* `Клавиша [D]` — удалить последнюю построенную стену.
* `Клавиша [R]` — полный сброс всей эволюции и очистка поля.
* `Клавиша [Пробел]` — ускоренный просчет популяции на 10 поколений вперед.
* `Клавиша [F]` — ускоренный просчет на 50 поколений вперед.
* `Клавиша [G]` — ускоренный просчет на 200 поколений вперед.
* `Клавиша [ESC]` — выход из симулятора.

## Системные требования и установка

Для запуска проекта вам понадобится Python 3.8+ и несколько библиотек.

### 1. Клонирование репозитория
```bash
git clone https://github.com/ВАШ_ПРОФИЛЬ/beevolution.git
cd beevolution
```

### 2. Установка зависимостей
Рекомендуется установить библиотеки `pygame`, `numpy` и `numba`:
```bash
pip install pygame numpy numba
```

### 3. Запуск игры
```bash
python main.py
```

## Структура проекта

* `main.py` — точка входа, игровой цикл, обработка событий ввода и рендеринг.
* `config.py` — конфигурационный файл с настройками разрешения, физики, параметров ГА и цветов.
* `population.py` — класс популяции, инициализация пчел и расчет функции приспособленности с использованием BFS.
* `physics.py` — физический движок игры и шаг эволюции (скрещивание/мутации), ускоренные с помощью Numba `@njit`.
* `ui.py` — отрисовка графических элементов интерфейса, отображение HUD и логика слайдов обучения.
* `presets.py` — готовые шаблоны карт-препятствий, масштабируемые под разрешение экрана.
* `assets/` — графические ресурсы проекта (фон, текстура цели и укушенного состояния).
